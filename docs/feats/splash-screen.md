# Splash Screen

How the boot / restart / shutdown splash works *now* (Plan 92).

A 2-second full-screen splash: the cat logo over a progress bar. The same screen
serves three purposes via a `Mode`, and it lives entirely in the **aether
presentation layer** — the kernel `Runtime` is never modified to draw it.

## What the user sees

- On UI boot, the desktop is pushed and then the splash appears on top of it: the
  cat logo with a progress bar filling underneath. After ~2s the bar completes and
  the splash pops, revealing the desktop already behind it.
- Tapping the **restart** tile in Mission Control shows the same splash for 2s,
  then the device restarts.
- The splash swallows all input while it runs — buttons do nothing during the 2s.
- It is full-screen on a black background, regardless of board resolution: the
  logo is scaled to fit the screen and the logo + bar are centred as a group.

## How it works

### The screen — `SplashScreen` (3 modes)

`nema::SplashScreen` (`firmware/core/include/nema/screens/splash_screen.h:11`) is a
`ComponentScreen` with `fullscreen() == true`
(`firmware/core/include/nema/screens/splash_screen.h:24`) and an empty `onAction()`
that swallows input (`firmware/core/include/nema/screens/splash_screen.h:20`).

`enum class Mode { Boot, Restart, Shutdown }`
(`firmware/core/include/nema/screens/splash_screen.h:13`) selects what happens when
the timer elapses. The duration is `kSplashMs = 2000`
(`firmware/core/src/screens/splash_screen.cpp:13`).

`tick()` drives the bar from elapsed wall-clock time and, on completion, performs
the mode's action (`firmware/core/src/screens/splash_screen.cpp:53`):
- `Mode::Boot` → `rt_.view().goBack()` — pops the splash to reveal the desktop.
- `Mode::Restart` → `rt_.requestRestart()`.
- `Mode::Shutdown` → `rt_.requestShutdown()`.

### Drawing — fit-scaled logo + progress bar

`draw()` (`firmware/core/src/screens/splash_screen.cpp:23`) clears to black, then:
- Scales the logo to the **logical** resolution keeping its 2:1 aspect: target
  width ≤ 4/5 of the canvas, target height ≤ 1/2, with small minimums
  (`firmware/core/src/screens/splash_screen.cpp:29`).
- Blits it via `nema::shell::blitScaledMask`
  (`firmware/core/src/screens/splash_screen.cpp:42`) — the **same scaler the
  desktop wallpaper uses**.
- Centres the logo + bar as one block on both axes
  (`firmware/core/src/screens/splash_screen.cpp:35`), then draws a rounded outline
  bar below the logo and fills it proportional to `progress_`
  (`firmware/core/src/screens/splash_screen.cpp:46`).

### The logo asset

`kSplashLogo` is a 1-bit, 128×64 mask
(`firmware/core/include/nema/assets/splash_logo.h:2`, referenced as `kLogoW=128,
kLogoH=64` in `firmware/core/src/screens/splash_screen.cpp:12`). It is generated by
`tools/asset_gen` from `tools/asset_gen/assets/splashscreen.png` with the
`--invert` option, because the art is light-on-dark: `png2c` normally treats
`gray < threshold` as ON, but `invert` flips it to `gray >= threshold`
(`tools/asset_gen/src/png2c.ts:37`). The config entry that makes this reproducible
is in `tools/asset_gen/asset_gen.config.json:2` (`"sym": "kSplashLogo",
"invert": true`).

### Architecture — owned by aether, not the kernel

The boot splash is pushed by `aether::bootSplash(rt)`
(`firmware/aether/src/boot_display.cpp:60`, declared in
`firmware/aether/include/aether/boot.h:29`). The `SplashScreen` instance is a
function-static owned by aether, not by any board or target.

Each target's `main` calls it **after** pushing the desktop, so the splash sits on
top and reveals the desktop when it pops:
- `firmware/targets/skyrizz-e32/main/main.cpp:42`
- `firmware/targets/wasm/main.cpp:39`

The restart splash is owned by the call site too: `MissionControlScreen` holds a
`SplashScreen` member constructed with `Mode::Restart`
(`firmware/core/src/screens/mission_control_screen.cpp:30`), and its restart tile
navigates to it (`firmware/core/src/screens/mission_control_screen.cpp:215`); the
splash then calls `requestRestart()` itself — there is **no kernel shutdown hook**.

By design, CLI/remote reboots are kernel paths and **skip** the splash entirely
(`firmware/core/src/screens/mission_control_screen.cpp:217`).
